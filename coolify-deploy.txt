Coolify — redeploy „one click” pentru mail-api (compose)
=========================================================

1) În Coolify: New Resource → Docker Compose.

2) Conectează repo-ul Git, branch (ex. main), base directory = rădăcina repo-ului.

3) Fișier Compose: compose.yaml

4) Environment Variables — ce trebuie completat (lista completă + comentarii):
      Fișier din repo: coolify-env.template
      Copiază liniile KEY=value în Coolify → Environment (sau import .env dacă există).

   Obligatoriu:
      ALLOWED_ORIGINS — ex. https://highvibescreation.com,https://www.highvibescreation.com
      (fără asta, docker compose refuză pornirea — mesaj de eroare la variabilă lipsă)

   De obicei la fel ca în template:
      SMTP_HOST=host.docker.internal
      SMTP_PORT=587
      SMTP_USER / SMTP_PASS — doar dacă Postfix cere auth

   Generează parolă aleatoare (opțional, pentru SMTP sau note proprii):
      node scripts/gen-coolify-secrets.js
      Copiază output-ul în Coolify; NU în git.

   Notă: Coolify afișează variabilele folosite în compose (${...}). Poți adăuga
   manual orice cheie din coolify-env.template dacă nu apare — același efect.

5) Deploy / Redeploy → build mail-api/Dockerfile, healthcheck GET /health.

6) Cloudflare: A/CNAME pentru subdomeniul API (ex. mail-api).

7) index.html: MAIL_API_BASE = URL public HTTPS al API (fără / final).

Auto-deploy: activează în Coolify deploy la push pe branch dacă vrei.
